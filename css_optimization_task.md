# Задача по оптимизации CSS проекта

## Общая цель:
Провести полную оптимизацию CSS-файлов проекта, вынести неиспользуемые стили в отдельный файл, устранить конфликты, дубликаты и похожие стили, а также составить карту изменений по страницам.

## Задача: Анализ и оптимизация всех CSS-файлов проекта

### 1. Поиск всех CSS-файлов
- Найти все CSS-файлы в проекте
- Составить полный список всех файлов
- Определить структуру и зависимости между файлами

### 2. Анализ и устранение конфликтов
- Провести аудит всех CSS-стилей
- Выявить конфликтующие стили, которые могут нарушать отображение элементов
- Устранить конфликты, приоритизируя используемые стили
- Убедиться, что стили не конфликтуют с общими стилями

### 3. Поиск и устранение дубликатов
- Найти дублирующиеся стили в разных файлах
- Объединить дублирующиеся стили в один
- Удалить дубликаты и перенести в файл неиспользуемых стилей

### 4. Поиск и оптимизация похожих стилей
- Найти похожие стили с небольшими различиями
- Объединить похожие стили в более общие классы
- Использовать CSS-переменные для повторяющихся значений

### 5. Составление карты изменений
- Для каждой страницы проекта составить карту изменений
- Указать, какие стили были изменены, удалены или добавлены
- Проверить отображение каждой страницы после оптимизации

### 6. Перенос неиспользуемых стилей
- Все неиспользуемые, устаревшие или дублирующие стили перенести в файл `unused_styles.css`
- Файл для переноса: `/workspace/styles/unused_styles.css`
- Добавить комментарии к каждому блоку стилей с объяснением, почему они не используются

## Структура проекта:
- `/workspace/css/` - основная директория CSS-файлов
- `/workspace/css/base/` - базовые стили (переменные, сброс, типографика)
- `/workspace/css/components/` - стили компонентов
- `/workspace/css/utils/` - утилитарные стили

## План выполнения:

### Этап 1: Анализ структуры
1. Создать список всех CSS-файлов
2. Определить зависимости между файлами
3. Найти используемые и неиспользуемые файлы

### Этап 2: Анализ содержимого
1. Провести аудит каждого CSS-файла
2. Найти все CSS-классы и селекторы
3. Сопоставить с HTML-файлами проекта

### Этап 3: Поиск конфликтов
1. Найти конфликтующие селекторы
2. Определить приоритеты
3. Устранить конфликты

### Этап 4: Удаление дубликатов
1. Найти дублирующиеся стили
2. Объединить дубликаты
3. Перенести в unused_styles.css

### Этап 5: Оптимизация похожих стилей
1. Найти похожие стили
2. Создать общие классы
3. Использовать CSS-переменные

### Этап 6: Тестирование
1. Проверить каждую страницу проекта
2. Убедиться в корректном отображении
3. Составить карту изменений

### Этап 7: Документирование
1. Записать все изменения в карту
2. Добавить комментарии в unused_styles.css
3. Создать отчет о проделанной работе

## Ожидаемый результат:
- Все неиспользуемые стили вынесены в `/workspace/styles/unused_styles.css`
- Конфликты устранены
- Дубликаты устранены
- Похожие стили оптимизированы
- Карта изменений для каждой страницы создана
- Проект корректно отображается на всех страницах
