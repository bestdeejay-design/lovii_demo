# Отчет об анализе и устранении конфликтующих CSS-стилей

## 1. Анализ текущего состояния

### 1.1. Инвентаризация CSS-файлов
- `/workspace/css/` - основная директория стилей
  - `/base/` - базовые стили (переменные, сброс, типографика)
  - `/components/` - компонентные стили
    - `buttons.css` - стили кнопок по БЭМ
    - `hero_non_bem.css` - стили героя (не БЭМ)
    - `categories_non_bem.css` - стили категорий (не БЭМ)
    - `navigation-icons_non_bem.css` - стили иконок навигации (не БЭМ)
    - `header.css` - пустой файл
    - и другие
  - `/utils/` - утилиты
  - `main.css` - главный файл импорта
  - `style.css` - стили шапки сайта

- `/workspace/styles/` - дополнительная директория стилей
  - `unused_styles.css` - неиспользуемые стили
  - `actually_unused_styles.css` - действительно неиспользуемые стили
  - `main_menu_styles.css` - стили главного меню
  - `main.css` - дублирующий файл

### 1.2. Используемые и неиспользуемые классы
- Используемые в HTML: `hero`, `hero-content`, `hero-image`, `categories-grid`, `category-card`, `nav-menu`, `nav-item`, `profile-menu` и др.
- Неиспользуемые в HTML (BEM-классы): `hero__content`, `categories__grid`, `header__nav-menu`, и др.

## 2. Схема подключения CSS к шаблонам и страницам

### 2.1. Подключение стилей
- В `main.css` импортируются:
  - Базовые стили
  - `style.css` (шапка сайта)
  - Компоненты (включая non_bem файлы)
  - Утилиты

### 2.2. Соответствие стилей и страниц
- `hero_non_bem.css` - используется на страницах с секцией героя
- `categories_non_bem.css` - используется на страницах с категориями
- `navigation-icons_non_bem.css` - используется в шапке сайта

## 3. Идентифицированные конфликты

### 3.1. Конфликты между BEM и не-BEM классами
- Определены BEM-классы (например, `.hero__content`) которые не используются в HTML
- Используются не-BEM классы (например, `.hero-content`) которые определены в non_bem.css файлах
- BEM-классы дублируют функциональность не-BEM классов

### 3.2. Дублирующиеся стили
- Схожие стили для кнопок, карточек, навигации в разных файлах
- Повторяющиеся паттерны оформления

## 4. Рекомендации по устранению конфликтов

### 4.1. Удаление неиспользуемых BEM-классов
- Удалить BEM-классы, которые не используются в HTML-файлах
- Перенести их в файл unused_styles.css если еще не перенесены

### 4.2. Оптимизация оставшихся стилей
- Объединить дублирующиеся стили
- Убедиться, что используемые стили работают корректно
- Привести структуру к единообразному виду

## 5. План действий

1. Проверить и дополнить перенос неиспользуемых BEM-классов в unused_styles.css
2. Убедиться, что все используемые классы определены корректно
3. Оптимизировать структуру CSS-файлов
4. Протестировать отображение всех страниц
