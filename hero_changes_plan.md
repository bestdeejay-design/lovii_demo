# План вносимых изменений для секции hero

## Цель
Устранить дублирование стилей, улучшить консистентность отображения, оптимизировать производительность и улучшить пользовательский опыт.

## Этапы реализации

### Этап 1: Удаление дублирующихся стилей
- Удалить стили hero из `/workspace/styles/main_menu_styles.css`
- Проверить, что все HTML-файлы используют правильные CSS-файлы

### Этап 2: Улучшение структуры CSS
- Добавить специфические классы для разных типов hero-секций
- Создать систему модификаторов для различных вариантов hero-секций
- Оптимизировать CSS-правила для лучшей производительности

### Этап 3: Улучшение HTML-структуры
- Обеспечить единообразие структуры hero-секций во всех файлах
- Добавить атрибуты для лучшей доступности

### Этап 4: Оптимизация изображений
- Проверить и при необходимости оптимизировать изображения
- Добавить альтернативные изображения для разных устройств

### Этап 5: Тестирование
- Проверить отображение на разных устройствах
- Проверить работу переключения тем
- Проверить производительность

## Подробные изменения

### 1. Удаление дублирующихся стилей
- Удалить все стили, связанные с `.hero`, из `/workspace/styles/main_menu_styles.css`
- Оставить в этом файле только действительно специфичные для меню стили

### 2. Модификация hero.css
- Добавить модификаторы для разных типов hero-секций:
  - `.hero--about` для about.html
  - `.hero--pvz` для pvz.html
  - `.hero--business` для business.html
  - и т.д.

### 3. Обновление HTML-файлов
- Заменить встроенные стили в pvz.html на CSS-классы
- Обеспечить единообразие структуры

### 4. Оптимизация подключения CSS
- Проверить, что все HTML-файлы подключают только необходимые CSS-файлы
- Удалить дублирующие подключения

## Ожидаемые результаты
- Устранение дублирования кода
- Улучшение производительности за счет уменьшения объема CSS
- Повышение консистентности дизайна
- Лучшая поддерживаемость кода
