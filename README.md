# Сайт «Лови» - Техническая документация

## Описание проекта
Полностью автономный, кликабельный B2B-сайт для проекта «Лови» (SaaS + оператор лояльности), который:
- работает локально (без интернета),
- не использует внешние CDN (все ресурсы — в репозитории),
- поддерживает адаптивность и смену тем (тёмная — по умолчанию),
- построен на шаблонах (DRY-подход),
- соответствует UI/UX лучших мировых медиа (BBC, CNN, The Guardian, Amazon),
- легко поддерживается и расширяется без разрушения структуры.


## Особенности реализации

### 1. Локальные ресурсы (без CDN)
- Шрифт Inter подключен локально через `@font-face` из `/assets/fonts/`.
- Иконки используются SVG из `/assets/icons/`.
- Все CSS/JS файлы локальные.

### 2. Система шаблонов
- Шапка, футер, мобильное меню подгружаются из `/templates/` через `templates.js`.
- Каждая страница содержит:
  - `<div id="dynamic-header"></div>`
  - `<div id="dynamic-footer"></div>`
  - `<div id="dynamic-mobile-nav"></div>`

### 2.1 Мобильная нижняя навигация
- Единый шаблон `/templates/mobile-nav.html` используется на всех страницах
- Автоматически отображается только на мобильных устройствах (ширина экрана ≤ 1023px)
- Включает в себя основные разделы: Продукты, О нас, Профиль, Вход
- Содержит выпадающее меню в разделе Профиль с настройками темы
- Интегрирована с системой переключения тем
- Поддерживает асинхронную загрузку без перезагрузки страницы
- Включает в себя полный футер сайта с основными разделами (Продукты, Компания, Поддержка, Аккаунт)

### 3. Темы: тёмная (по умолчанию) + светлая
- Реализовано через CSS-переменные в `themes.css`
- Текущая тема сохраняется в `localStorage`.
- Переключатель в шапке и мобильном меню.

### 4. Адаптивность (mobile-first)
- Поддержка:
  - Мобильные: 320–767px
  - Планшеты: 768–1023px
  - Десктоп: 1024px+
- Мобильное меню внизу экрана, как в Amazon/Instagram (для управления одной рукой).
- Все элементы не менее 48×48px (для удобства тапа).

### 5. UI/UX по примеру медиа-гигантов
- **BBC / CNN**: чёткая иерархия, высокий контраст CTA, минимализм.
- **Amazon**: карточки товаров с превью, быстрые действия.
- **The Guardian**: плавные переходы, микроанимации при hover/focus.
- **LinkedIn**: персонализированные блоки с иконками и временем.

## Требования к качеству кода

### CSS
- БЭМ-методология
- Без использования `!important`
- Все стили через переменные

### JavaScript
- Без глобальных переменных
- Модульная структура
- Поддержка шаблонов

### HTML
- Семантические теги (`<nav>`, `<main>`, `<section>`)
- ARIA-атрибуты для доступности

## Проверка и поддержка

### Перед коммитом:
- Открыть `file://` в браузере → убедиться, что всё работает без интернета.
- Проверить, что темы переключаются и сохраняются.
- Убедиться, что мобильное меню появляется внизу на <768px.
- Документирование: изменения сначала в `pages.md`, после реализации — отчёт в `pages.md`.

## Источники вдохновения
- **CTA и формы**: BBC, The Guardian → минимализм, высокий контраст.
- **Адаптивное меню**: Amazon, Instagram → фиксированное снизу на мобилках.
- **Анимации**: LinkedIn, Twitter → микроинтеракции при обновлении.
- **Доступность**: WCAG 2.1 AA — контраст ≥ 4.5:1, навигация с клавиатуры.

## Интеграция адаптивного футера

В рамках проекта был разработан и интегрирован адаптивный футер с четырьмя колонками, вдохновленный лучшими практиками крупных медиа брендов. Футер включает:

- Информацию о компании
- Быстрые ссылки
- Полезные ресурсы
- Контактную информацию и социальные иконки

Футер полностью адаптирован под все устройства и соответствует профессиональным стандартам веб-дизайна. Стили были добавлены в существующий файл main.css, а HTML-разметка интегрирована в index.html.
