# Исправление проблемы со сменой темы на адаптивной версии

## Проблема
При работе с мобильной версией сайта возникали проблемы с синхронизацией состояния переключателей темы. В шаблоне есть два переключателя темы:
- Один в десктопной навигации (отображается на больших экранах)
- Один в мобильном меню (отображается в off-canvas меню на маленьких экранах)

Проблема заключалась в том, что при переключении темы с помощью одного из переключателей, состояние другого могло не обновляться должным образом на мобильных устройствах.

## Причина
В исходном коде использовалось прямое манипулирование стилями элементов `.toggle-knob` через JavaScript:
```javascript
knob.style.transform = isForest ? 'translateX(18px)' : 'translateX(0)';
```

Это могло приводить к проблемам синхронизации состояния между разными переключателями, особенно на мобильных устройствах.

## Решение
1. Изменили логику обновления состояния переключателей - теперь вместо прямого изменения стилей через JavaScript, мы устанавливаем CSS-классы на родительские элементы переключателей
2. Добавили дополнительное CSS-правило для корректной обработки состояния переключателей, когда класс `theme-forest` применяется непосредственно к элементу `.theme-toggle`

## Изменения
### JavaScript (`/workspace/js/main.js`)
- Обновлена функция `updateToggleKnobs` для установки классов на элементы `.theme-toggle` вместо прямого изменения стилей
- Теперь все переключатели синхронизируются корректно при изменении темы

### CSS (`/workspace/css/style.css`)
- Добавлено дополнительное правило `.theme-toggle.theme-forest .toggle-knob` для корректной обработки состояния переключателей

## Результат
Теперь переключение темы работает корректно на всех устройствах, и состояние всех переключателей синхронизировано.
